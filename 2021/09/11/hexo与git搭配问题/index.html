Hexo github action部署修改配置

之前用Hexo部署的时候一直把源代码放在公共仓库的。今天看到大家都把代码放到私有仓库再透过github Action来部署。Hexo的具体的教程。按照教程可以轻松的配置出来。

问题在于当我使用了两个仓库分别放源码和网页文件时，出现了在使用hexo deploy 和hexo backup命令时始终push到一个仓库的问题。经过几次测试。解决方案如下，

1.要在git里面新增一个额外的remote连接。git remote add origin <remote-url> .

2.修改_config.yml

里面的名字是在git建立远程链接时设定的。经过这个小小的修改，后续就可以愉快的deploy和back up了。
